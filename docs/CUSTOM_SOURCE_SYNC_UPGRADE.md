# 自定义音源同步升级说明

本文记录 `Tingfe/lx-music-mobile` 对 LX Music 移动端的定制改动，并说明如何升级和验证。

## 此 Fork 的改动

- 新增 `userApi` 同步功能：同步「设置 → 自定义源」中导入的音源元信息和完整 JavaScript 脚本。
- 同一同步账号的在线设备在音源添加、删除或修改后会收到实时更新。
- 接收到更新后，如果该音源正被选为当前自定义源，应用会重新加载其脚本。
- 新增「设置 → 数据同步 → 控制在线车机」入口：仅当已连接同步服务且车机在线时才显示可控设备；可切歌、播放/暂停，并让车机播放手机当前歌曲。
- 遥控指令不写入同步数据，也不会发送给离线车机；手机版没有登记为受控车机，因此不会影响普通手机播放与同步功能。
- `master` 分支推送会构建可独立运行的测试 APK，不需要 Android 签名密钥，并以 GitHub `Pre-release` 形式发布。
- 应用内「检查更新」读取 `Tingfe/lx-music-mobile` 的正式 Release，不会跳转或下载原作者仓库的版本；它会按当前安装包自动匹配 `mobile` 或 `car` APK。

歌单和不喜欢列表的既有同步功能未改变。旧服务端不支持 `userApi` feature 时，移动端会继续使用旧功能，但不会同步音源。

## 升级顺序

1. 先将同步服务端升级到 `v2.1.11` 或更高版本，具体见服务端仓库的升级说明。
2. 将本移动端升级到包含本功能的提交或之后的测试 APK。
3. 在所有需要同步的设备上连接同一个同步服务和账号。

首次连接到空服务端时，第一台连接的设备会把本机音源写入服务端。服务端已有音源数据时，服务端版本会覆盖刚连接设备的本地音源，避免旧设备意外覆盖较新的配置。

## 获取和安装测试 APK

1. 打开本仓库的 **Releases** 页面，选择最新的 `Pre-release` 测试版本。
2. 下载唯一的 `universal.apk`；它适用于大多数 Android 设备。

同一文件也会保留在对应 `Build Test APK` Actions 任务的 Artifacts 中。

## 发布正式版本与应用内升级

测试预发布只供手动验证，不会被应用内升级检测到。需要发布可由应用内升级的正式版本时：

1. 将 `package.json` 的 `version` 与 `versionCode` 升级到新版本。
2. 提交并推送该版本的代码。
3. 创建并推送同名 Git tag，例如版本为 `1.8.5` 时执行 `git tag v1.8.5`、`git push origin v1.8.5`。
4. GitHub Actions 会构建手机 universal APK，并创建正式 Release。应用内检查更新会从该 Release 下载该 APK。

测试 APK 使用调试签名，但已内置 JavaScript bundle，因此无需连接 Metro 开发服务器。若设备上安装了使用不同签名构建的同包名应用，Android 可能要求先卸载旧应用才能安装。

服务端升级、恢复数据目录或调整同步用户名后，如果旧设备密钥不再被服务端接受，应用会自动要求重新输入连接码；无需清除应用数据、歌单或缓存。

## 跨端设置同步

新版会同步主题、语言、播放模式、列表与搜索显示偏好等跨设备体验设置。同步地址、连接码、设备密钥、存储路径、缓存、下载内容和音频硬件项始终只保存在当前设备。

## 验证步骤

1. 在设备 A 的「设置 → 自定义源」导入一个已验证可用的 JS 音源。
2. 确保设备 A 已连接同步服务，等待同步状态显示已连接。
3. 在设备 B 连接同一账号；音源列表应自动出现该音源。
4. 在设备 B 选择该自定义源，并用搜索或播放功能确认脚本已成功加载。
5. 在设备 A 删除或重新导入音源，确认设备 B 的列表同步更新。
6. 车机保持在线并连接同一账号后，在手机「设置 → 数据同步 → 控制在线车机」中测试上一首、下一首、播放/暂停和“在车机播放手机当前歌曲”。

## 安全说明

自定义音源是可执行 JavaScript。同步功能会将脚本完整复制到同一同步账号的设备上，因此仅应导入你信任的音源，并仅在自己控制的私有同步服务和账号中使用。
